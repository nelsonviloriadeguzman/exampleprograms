Identifier Format encompasses several widely accepted best practices that promote readability, maintainability, and consistency across codebases.

PascalCase used typically for class name, interfaces, enums and type names. Capitalize the first letter of each word without spaces or underscores.

UPPERCASE should be used for all characters in variables designated as constants or immutable.

camelCase used typically for variables, functions, and method names. Capitalize the first letter of each word except the first word, without spaces or underscores.

Code: 

  
    public class Main {

        static class DailyGreetings {
      
          public static final String IMPORTANT_MESSAGE = "Safety First!";
          
          public static void morningGreetings() {
      
            String additionalMessage = "Eat your breakfast!";
            System.out.println("Good Morning!");
            System.out.println(additionalMessage);
            System.out.println(IMPORTANT_MESSAGE);
            
          }
        }
        
        public static void main(String[] args) {
      
          DailyGreetings.morningGreetings();
          
        }
      
    }

Concept Description
DailyGreetings class name example for PascalCase.
IMPORTANT_MESSAGE constant name example for UPPERCASE.
morningGreetings method or function name example for camelCase.
additionalMessage variable name example for camelCase.
